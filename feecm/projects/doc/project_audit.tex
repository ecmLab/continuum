\section{Scope and method}
Each input file under \code{feecm/projects/<app>/<project>/} (top-level and
one folder deeper where projects nest) was run through the corresponding
app executable with \code{-{}-check-input}. This validates the input file's
syntax, parser/factory registrations, mesh/file references, and basic
integrity, but does \emph{not} time-step the simulation. An input that
passes \code{-{}-check-input} may still fail at solve time; a failing
input is definitely broken at parse/setup.

The four apps and their executables:
\begin{itemize}
  \item \code{ecBeta} $\to$ \code{ec\_beta/ec\_beta-opt}
  \item \code{ecmTest} $\to$ \code{ecm\_test/ecm-opt}
  \item \code{eel} $\to$ \code{eel/eel-opt}
  \item \code{tecmTest} $\to$ \code{tecm\_test/tecm\_test-opt}
\end{itemize}

\code{Invalid value} & 1 & A parameter has an invalid expression or symbolic value (e.g.\ a string where a Real is expected). \\
