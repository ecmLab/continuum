\section{Scope and method}
Every input file under \code{feecm/projects/<app>/} was run through the corresponding app executable with \code{-{}-check-input}. This validates input syntax, parser/factory registrations, mesh/file references, and basic integrity, but does \emph{not} time-step the simulation. An input that passes \code{-{}-check-input} may still fail at solve time; a failing input is definitely broken at parse/setup.

As of this audit the repository has \textbf{two} apps (the former \code{ec\_beta} and \code{eel} apps were consolidated into \code{ecm\_test} and \code{tecm\_test} respectively and removed):
\begin{itemize}
  \item \code{ecmTest} $\to$ \code{ecm\_test/ecm-opt}
  \item \code{tecmTest} $\to$ \code{tecm\_test/tecm\_test-opt} (now also hosts the former \code{projects/eel/} work)
\end{itemize}
