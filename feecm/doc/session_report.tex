\section{Why this report exists}

Between 2026-05-06 and 2026-05-09 the \code{feecm/} repository went
through a substantial maintenance pass: branch reconciliation, source
integration of contributions from a long-divergent collaborator branch
(\code{AshokG}), a project-tree restructure, MOOSE-deprecation fixes
across hundreds of input files, and finally the consolidation of the
\code{ec\_beta} app into \code{ecm\_test} so that the former could be
deleted.

This document is a self-contained recall aid: what was done, why,
where the artifacts live, and what remains open.

\section{Top-of-mind: what changed structurally}

\subsection{The four-app layout collapsed to three}

\textbf{Before (start of session):}
\begin{tightitemize}
  \item \code{ec\_beta/} --- electrochemistry-only app
  \item \code{ecm\_test/} --- electro-chemo-mechanics
  \item \code{eel/} --- battery physics from literature (Argonne 2023)
  \item \code{tecm\_test/} --- thermal-electro-chemo-mech, forked from eel
\end{tightitemize}

\textbf{After:}
\begin{tightitemize}
  \item \code{ecm\_test/} --- absorbed all 74 of \code{ec\_beta}'s registered classes; \code{ecm-opt} is now a strict superset of the former \code{ec\_beta-opt}
  \item \code{eel/} --- unchanged scope; gained 5 \code{*Scaled} non-dim materials from Ashok and small revisions to \code{ChargeTransferReaction}/\code{MassDiffusion}
  \item \code{tecm\_test/} --- unchanged scope
  \item \code{ec\_beta/} --- \textbf{deleted} (commit \code{b50b8738})
\end{tightitemize}

\subsection{Project tree rationalized}

Input files (\code{.i}) used to live inside each app under \code{problems/} or \code{examples/}.
The user moved all of them to a top-level \code{projects/} tree to
separate science work from app source. The current layout:

\begin{verbatim}
feecm/
  ecm_test/         <- app
  eel/              <- app
  tecm_test/        <- app
  patchMoose2Feecm/ <- MOOSE framework patches (manual apply)
  doc/              <- repo-wide docs (this report lives here)
  projects/         <- science workspace
    ecmTest/    (43 sub-projects, includes former ecBeta)
    eel/        (13 sub-projects)
    tecmTest/   (3 sub-projects)
    paper_shafee_cathode/
    doc/        <- per-project audit reports
\end{verbatim}

\subsubsection*{The CurrentDensity conflict resolution}

\code{ec\_beta/src/auxkernels/CurrentDensity.C} (last touched 2024-09-03) used
prefactor $\times 10$ (cm-coordinate convention). \code{ecm\_test/src/auxkernels/CurrentDensity.C}
(last touched 2025-06-01) used $\times 10000$ (mm-coordinate convention). Both
documented conductivity in \code{mS/cm}. We kept ecm\_test's newer
$\times 10000$ version because: (a) it's chronologically the user's most recent
expression of intent; (b) it matches the mm-coordinate convention used in
current battery-cell projects.

\textbf{Caveat:} 24 ecBeta-side inputs that referenced \code{type = CurrentDensity}
were calibrated against the $\times 10$ version. After migration they will produce
values 1000$\times$ larger. If any of those values feed published figures or
absolute-magnitude comparisons, verify before trusting outputs.
